RepoHeader

The sticky-at-top block that identifies the repository a user is looking at. It composes several smaller chip components (OwnerChip, RepoChip, RepoVersionChip), a follow button, and a Manage menu for users with write access — then shows the repo name and canonical URL in large type underneath.

Anatomy

  1. Container — Paper with background surface.main, 16px padding, radius 10px 10px 0 0 (flush with tab strip below), no shadow.
  2. Chip row — left side: OwnerChip + RepoChip + (optional) RepoVersionChip. Right side: follow button + Manage menu button.
  3. Follow button — FollowActionButton rendered as an icon button.
  4. Manage menu — only shown when the current user has write access and the entity has a source_type or collection_type. Opens RepoManagementList.
  5. Title — repo name at 28px, weight 600, color surface.dark.
  6. Canonical URL — 14px in secondary.main, shown only when repo.canonical_url exists.

States

  • Read-only viewer — no Manage button, follow button still present.
  • Version switcher active — RepoVersionChip appears between RepoChip and the right side.
  • Repo version page — isVersion is true; Manage menu offers different options (e.g. "Edit version", "Release version") via RepoManagementList.
  • No canonical URL — subtitle is omitted entirely, no empty space reserved.

Props

PropTypeDescription
repoobjectThe repository or repository version entity.
ownerobjectUser or organization that owns the repo.
versionsarrayList of versions, passed to RepoVersionChip.
isVersionboolTrue when rendering on a version page — changes available Manage actions.
onVersionChangefnCallback when the user picks a different version. Presence triggers the RepoVersionChip render.
onCreateConceptClickfnFired from Manage menu → "Add concept".
onCreateMappingClickfnFired from Manage menu → "Add mapping".
onCreateVersionClickfnFired from Manage menu → "Create version".
onVersionEditClickfnFired from Manage menu → "Edit version" (version pages only).
onReleaseVersionClickfnFired from Manage menu → "Release version".
onDeleteRepoClickfnFired from Manage menu → "Delete".
