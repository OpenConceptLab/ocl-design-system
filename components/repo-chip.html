RepoChip

A compact, clickable reference to a repository (Source or Collection). Renders an icon, the repo's short code in bold, an optional version, and the repo type, with an optional RepoTooltip wrapper for hover details.

Anatomy

Source: designs/Chip and Tooltip.png

  1. Avatar — folder icon (filled or outlined), or a custom logo_url if the entity has one.
  2. Entity ID — the repo's short_code or id, bold, in black.
  3. Version — shown when the entity is a Source Version or Collection Version.
  4. Divider — a 3px circle in secondary.main, 80% opacity.
  5. Entity type — Source / Collection / Source Version etc., in secondary.main.
  6. Container — outlined chip, 4px radius, minimum width 95px, border surface.nv80.

Variants

Primary vs. secondary (background)

  • primary: background primary.95 — use for the main chip on a page (e.g. the active repo in the breadcrumb).
  • default: white background — use inside dense lists or when another chip on the row is primary.

Size

  • medium (default): 36px tall, 14px text, 18px icon.
  • small: 28px tall, 12px text, 16px icon.

Tooltip

  • Default: wrapped in RepoTooltip (hover to see repo summary).
  • noTooltip: render just the chip, no wrapper.
  • basicTooltip: simpler tooltip variant.

Filled icon

  • filled: uses the filled Folder icon instead of FolderOutlined. Reserved for the active/selected repo.

Props

PropTypeDefaultDescription
repoobject—Repository entity. Expects short_code, id, type, optional version, url, version_url, logo_url.
noTooltipboolfalseRender without the RepoTooltip wrapper.
basicTooltipboolfalseUse the basic tooltip variant.
filledboolfalseUse the filled folder icon.
…plus all props forwarded to BaseEntityChip: size, primary, hideType, hideRepoVersion, noLink, sx.

Usage

TODO — needs spec. Do/don't guidance hasn't been written yet. Draft candidates:
  • Use RepoChip wherever you're referencing a repo by name in running text or inside a chip row.
  • Don't stack more than 3 chips in a single horizontal row — use a Breadcrumbs component instead.
  • Prefer primary for the currently-focused repo on a page; use the default background elsewhere.
