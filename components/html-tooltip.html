HTMLTooltip

A styled MUI Tooltip that allows arbitrary HTML in the tooltip body. Used as the base for richer hover cards (RepoTooltip, ConceptTooltip, OwnerTooltip) where a plain text string isn't enough.

Anatomy

  1. Container — 285px fixed width, white background, 1px border surface.nv80, 5px radius, 12px padding.
  2. Shadow — layered box-shadow: 0 2px 3px 0 rgba(0,0,0,0.3), 0 6px 10px 4px rgba(0,0,0,0.15). This is a one-off (not MUI's elevation system) — do not change without coordinating with HTMLTooltip.jsx.
  3. Body — arbitrary React children. Default text color secondary.main, 12px.
  4. Arrow — disabled in HTMLTooltip even though the theme default is arrow: true. See theme override below.

Theme-level defaults

The app's theme (oclweb3/src/index.jsx) sets global defaults for all MUI Tooltip instances: 

MuiTooltip: {
  defaultProps: {
    arrow: true,
    leaveDelay: 300,
    TransitionComponent: Fade,
    TransitionProps: { timeout: 300 }
  }
}

These apply to regular <Tooltip> usage site-wide. HTMLTooltip overrides arrow back to false because the custom-bordered container doesn't pair well with a speech-bubble arrow. The 300ms fade still applies.

Usage

TODO — needs spec. Draft candidates:
  • Use HTMLTooltip when the hover content needs formatting (multiple lines, nested labels, entity chips).
  • Use a plain <Tooltip title="…" /> when a short text label is enough — get the arrow for free.
  • Don't nest an HTMLTooltip inside another HTMLTooltip.
  • Keep content under ~4 lines. Anything longer belongs in a popover or dialog.
